TCredit
역할별 입력 및 확인 절차

공헌 등록, 추천 의견, 승인 처리, 위원회 심의, 관리자 확인 화면을 실제 로컬 사이트 화면 캡처와 함께 정리한 운영 매뉴얼입니다.

기준 화면http://localhost:3000
작성일2026년 6월 8일
대상시범운영 사용자 및 운영자
본 문서는 현재 로컬 TCredit 사이트와 라이브 PostgreSQL 데이터를 기준으로 작성했습니다. 화면의 공헌 건수, 사용자 이름, 대기 건수는 운영 데이터 변경에 따라 달라질 수 있습니다.

빠른 시작

1. 계정 선택
상단 사용자 이름을 눌러 역할별 계정으로 전환합니다.
2. 공헌 입력
공헌자 계정에서 공헌 제목, 유형, 활동일, 관련 부서, Tier, 내용을 입력합니다.
3. 추천 확인
추천자로 지정된 사용자는 공헌 내역 화면에서 추천 의견을 입력합니다.
4. 승인 처리
승인권자는 추천 내용을 확인하고 점수, Tier, 의견을 입력한 뒤 승인 또는 반려합니다.
5. 심의/관리
위원회와 관리자는 이상 사례, 예산, 사용자 권한, 임계값을 확인합니다.
6. 결과 확인
공헌 내역과 운영 현황에서 상태와 Credit 반영 결과를 확인합니다.

공헌자: 공헌 입력

공헌자는 실제 기여 내용을 등록하고, 필요한 경우 동료 추천인을 지정합니다. 입력 완료 후 상태는 활동일과 추천인 지정 여부에 따라 달라집니다.

1. 메뉴 이동
좌측 메뉴에서 공헌 입력을 선택합니다.
2. 기본 정보 입력
공헌 제목, 공헌 유형, 활동일, 관련 부서, Effort Tier를 입력합니다. 활동일은 미래일이 아니어야 하며, 활동일 기준 30일 이내 입력 건은 추천 및 승인 흐름으로 이동합니다.
3. 공헌 내용 작성
기여 배경, 수행 내용, 협업 부서, 결과를 승인자가 판단할 수 있을 만큼 구체적으로 작성합니다.
4. 추천인 선택
추천이 필요한 경우 동료추천인 선택 목록에서 추천인을 고르고 추가를 누릅니다. 선택한 추천인은 제출 후 추천 의견 입력 대상이 됩니다.
5. 제출
오른쪽 산정 미리보기에서 예상 Credit을 확인한 뒤 제출을 누릅니다.
추천인을 선택하면 공헌은 먼저 추천 대기 흐름으로 이동합니다. 추천인이 의견을 저장하면 승인권자가 검토할 수 있는 승인 대기 상태로 넘어갑니다.

공헌자/추천자: 공헌 내역 확인 및 추천 의견

공헌 내역 화면은 본인의 공헌 상태를 확인하는 화면이면서, 추천자로 지정된 사람이 추천 의견을 입력하는 화면이기도 합니다.

1. 본인 공헌 확인
누적 승인 Credit, 입력 공헌 수, 추천 요청 수를 확인합니다.
2. 검색과 기간 필터
제목, 내용, 부서, 유형 검색어와 시작/종료일을 입력해 공헌 목록을 좁힙니다.
3. 상태 확인
각 공헌의 상태 배지를 확인합니다. 상태 옆 물음표 아이콘은 상태 의미와 다음 단계를 설명합니다.
4. 추천 의견 입력
추천 요청이 있는 경우 하단 추천 의견 입력 영역에 의견을 작성하고, 필요하면 비공개 의견으로 제출합니다.

상태 흐름

추천 대기추천인이 의견을 입력해야 하는 단계
승인 대기승인권자가 점수와 Credit을 검토하는 단계
승인Credit이 확정되어 내역에 반영된 상태
반려공헌 인정 대상이 아니거나 보완이 필요한 상태
입력기한 초과활동일 기준 30일을 초과해 일반 승인 대상에서 제외된 상태
위원회 검토이의신청, 비공개 의견, 이상 사례 검토 단계
초안작성 중이며 아직 제출되지 않은 상태
결과 확인운영 현황과 공헌 내역에서 최종 상태 확인

승인권자: 승인 처리

승인권자는 자신이 담당하는 관련 부서의 승인 대기 공헌을 검토합니다. 관리자는 전체 범위의 승인 대기 건을 볼 수 있습니다.

1. 승인 처리 메뉴 이동
좌측 승인 처리를 선택합니다. 처리 가능한 공헌이 있으면 가장 먼저 검토할 대상이 표시됩니다.
2. 공헌 내용과 추천 의견 확인
제목, 설명, 공헌자, 관련 부서, 활동일, 공개/비공개 추천 의견을 검토합니다.
3. 점수와 Tier 조정
참여, 성과, 효과 체크 여부와 최종 Tier를 조정합니다. 오른쪽 승인 산정 영역에서 최종 Credit을 확인합니다.
4. 승인/반려 의견 작성
판단 근거를 의견란에 입력합니다.
5. 최종 제출
승인 또는 반려 버튼을 누르면 확인 창이 뜹니다. 확인 후 결정이 저장됩니다.

승인권자: 승인자 조회

승인자 조회는 처리 현황을 요약하고, 승인 대기 건이나 처리 편차를 빠르게 확인하는 화면입니다.

승인율현재 계정이 처리한 승인/반려 결정 기준의 승인율을 보여줍니다.
발행 Credit승인 결정으로 누적 발행된 Credit을 보여줍니다.
참고 사항승인 대기 건수나 운영상 점검이 필요한 조건을 안내합니다.
승인 내역이미 승인 또는 반려한 건을 확인하고, 항목을 펼쳐 공헌 내용을 확인합니다.

위원회: 심의 및 표본감사 확인

위원회 역할은 일반 승인 화면과 별도로 심의 대기, 비공개 의견, 표본감사, 이의신청 대상을 확인합니다.

1. 심의 대기 확인
이상 징후, 반려 이의신청, 위원회 검토 대상 건수를 확인합니다.
2. 비공개 의견 확인
추천자가 비공개로 제출한 의견 수와 검토자료를 확인합니다.
3. 이상 사례 큐 확인
열린 이상 사례가 있으면 신호 유형, 심각도, 공헌 내용을 확인합니다.
4. 무작위 표본감사 확인
표본감사 대상 공헌과 Credit 규모를 확인합니다.

관리자: 사용자, 권한, 예산 확인

관리자는 전체 운영 환경을 확인합니다. 현재 화면은 관리 기능 입력보다는 사용자/권한과 예산 풀, 심의 임계값 확인에 초점이 있습니다.

1. 사용자 및 권한
사용자 이름, 사번, 소속, 직책, 역할을 확인합니다.
2. 예산 풀
운영 기간, 총 예산, 1 Credit 명목가치, 발행 가능 Credit을 확인합니다.
3. 심의 임계값
승인권자 편중, 반려 비율, 중복 승인 요청, 월말 승인 집중도 등 위원회 점검 조건을 확인합니다.
4. 역할 기반 점검
특정 부서의 승인 대기 공헌이 승인권자에게 보이지 않으면 해당 부서에 APPROVER 역할 사용자가 있는지 확인합니다.

운영 체크리스트

상황 확인할 화면 조치 기준
공헌 제출 후 승인권자에게 보이지 않음 공헌 내역, 승인 처리, 관리자 추천인이 지정된 건이면 추천 의견 제출 후 승인 대기로 이동합니다. 관련 부서와 승인권자 소속이 일치하는지도 확인합니다.
추천 의견 입력 영역이 비어 있음 공헌 내역 현재 사용자에게 요청된 추천 의견이 없거나, 이미 제출되어 대기 목록에서 사라진 상태입니다.
Credit이 예상과 다름 승인 처리, 공헌 내역 최종 Credit은 승인권자가 참여, 성과, 효과 점수와 최종 Tier를 조정한 결과입니다.
입력기한 초과 상태 표시 공헌 내역, 운영 현황 활동일 기준 30일을 초과한 입력 건은 일반 승인 대상이 아니라 unbillable로 분류됩니다.
위원회 검토가 필요한 경우 위원회 심의 비공개 의견, 이상 사례, 이의신청, 표본감사 대상을 확인하고 운영 정책에 따라 후속 처리합니다.
실제 운영 배포 환경에서는 계정 전환 드롭다운이 비활성화될 수 있습니다. 시범운영 또는 개발 환경에서는 역할별 검증을 위해 계정 전환 기능을 사용할 수 있습니다.
